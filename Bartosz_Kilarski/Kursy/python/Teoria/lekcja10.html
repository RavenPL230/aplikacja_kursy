1. Wprowadzenie do Garbage Collection:

Garbage Collection (GC) to mechanizm zarządzania pamięcią w Pythonie, który automatycznie usuwa nieużywane obiekty z pamięci. Python śledzi referencje do obiektów, a gdy dany obiekt nie jest już potrzebny (nie ma referencji do niego), zostaje automatycznie usunięty, aby zwolnić pamięć.

2. Referencje i zarządzanie obiektami w pamięci:

Każdy obiekt w Pythonie ma przypisaną liczbę referencji, które wskazują, ile razy dany obiekt jest używany. Gdy liczba referencji spada do zera, obiekt może zostać usunięty. Python automatycznie zarządza referencjami, ale warto zrozumieć ten mechanizm przy pracy z bardziej złożonymi strukturami danych.

3. deepcopy() vs copy():

Python oferuje dwie funkcje do kopiowania obiektów: copy() oraz deepcopy(). Oto ich różnice:

Funkcja copy():

Tworzy płytką kopię obiektu, co oznacza, że obiekt zewnętrzny zostaje skopiowany, ale obiekty zagnieżdżone są współdzielone. Przykład:

import copy

lista1 = [1, 2, [3, 4]]
lista2 = copy.copy(lista1)
lista2[2][0] = 99
print(lista1)

Funkcja deepcopy():

Tworzy głęboką kopię obiektu, co oznacza, że wszystkie elementy, zarówno zewnętrzne, jak i zagnieżdżone, są kopiowane, co pozwala na niezależność obiektów. Przykład:

import copy

lista1 = [1, 2, [3, 4]]
lista2 = copy.deepcopy(lista1)
lista2[2][0] = 99
print(lista1)
