1. Używanie przestrzeni nazw (package)

Przestrzenie nazw (ang. packages) w Javie służą do organizowania kodu w logiczne grupy, co ułatwia zarządzanie i unikanie konfliktów nazw. Oto jak zdefiniować przestrzeń nazw i używać jej w kodzie:

  • Definiowanie przestrzeni nazw:
    package nazwaPrzestrzeni;
  • Używanie przestrzeni nazw:
    import nazwaPrzestrzeni.NazwaKlasy;



/* Plik: Pracownik.java */
package pl.mojafirma;

public class Pracownik {
    String imie;
    
    public Pracownik(String imie) {
        this.imie = imie;
    }
    
    public void przedstawSie() {
        System.out.println("Czesc, nazywam sie " + imie);
    }
}


/* Plik: Test.java */
package pl.mojafirma;

import pl.mojafirma.Pracownik;

public class Test {
    public static void main(String[] args) {
        Pracownik pracownik = new Pracownik("Anna");
        pracownik.przedstawSie();
    }
}

2. Definiowanie i używanie modułów

W Javie moduły są częścią systemu modułów wprowadzonego w Javie 9. Moduły pozwalają na organizowanie kodu w większe jednostki. Oto jak definiować i używać modułów:

  • Definiowanie modułu:
    module nazwaModulu { /* eksportowane pakiety */ }
  • Używanie modułu:
    requires nazwaModulu;


/* Plik: module-info.java */
module com.example.mymodule {
    exports com.example.mypackage;
}


/* Plik: module-info.java */
module com.example.mainmodule {
    requires com.example.mymodule;
}

W powyższych przykładach:

  • Plik: module-info.java w katalogu com/example/mymodule definiuje moduł com.example.mymodule, który eksportuje pakiet com.example.mypackage.
  • Plik: module-info.java w katalogu com/example/mainmodule definiuje moduł com.example.mainmodule, który wymaga modułu com.example.mymodule.

3. Importowanie bibliotek standardowych i zewnętrznych

Biblioteki standardowe są częścią JDK i można je zaimportować bezpośrednio do projektu. Biblioteki zewnętrzne można dodać do projektu poprzez menedżer zależności, taki jak Maven lub Gradle:

  • Importowanie bibliotek standardowych:
    import java.util.ArrayList;
  • <dependency> - Rozpoczyna definicję zależności w pliku `pom.xml` Mavena.
  • <groupId>com.example</groupId> - Identyfikator grupy, do której należy biblioteka. Jest to zazwyczaj unikalna nazwa organizacji lub firmy.
  • <artifactId>external-lib</artifactId> - Unikalna nazwa samej biblioteki (artefaktu). W tym przypadku jest to `external-lib`.
  • <version>1.0.0</version> - Numer wersji biblioteki, który wskazuje, której wersji używamy.
  • </dependency> - Kończy definicję zależności.


/* Plik: ListaPrzykladowa.java */
import java.util.ArrayList;

public class ListaPrzykladowa {
    public static void main(String[] args) {
        ArrayList lista = new ArrayList<>();
        lista.add("Element1");
        lista.add("Element2");
        for (String element : lista) {
            System.out.println(element);
        }
    }
}

4. Tworzenie i używanie własnych bibliotek

Tworzenie własnych bibliotek polega na definiowaniu klas i metod, które można wielokrotnie używać w różnych projektach. Oto przykład tworzenia i używania własnej biblioteki:



/* Plik: MojaBiblioteka.java */
package com.mojafirma.mylib;

public class MojaBiblioteka {
    public static void powiedzHello() {
        System.out.println("Hello from my library!");
    }
}


/* Plik: TestBiblioteki.java */
import com.mojafirma.mylib.MojaBiblioteka;

public class TestBiblioteki {
    public static void main(String[] args) {
        MojaBiblioteka.powiedzHello();
    }
}
