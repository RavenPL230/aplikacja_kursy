1. Instrukcje warunkowe

Instrukcje warunkowe umożliwiają podejmowanie decyzji na podstawie spełnienia określonych warunków. W zależności od tego, czy warunek jest prawdziwy, wykonuje się odpowiednie bloki kodu. Oto podstawowe instrukcje warunkowe w Javie:

  • Instrukcja if:
    if (warunek) { /* kod */ }
  • Instrukcja else if:
    else if (innyWarunek) { /* kod */ }
  • Instrukcja else:
    else { /* kod */ }


    • Przykład: 
public class WarunkiExample {
    public static void main(String[] args) {
        int liczba = 10;

        if (liczba > 0) {
            System.out.println("Liczba jest dodatnia");
        } else if (liczba < 0) {
            System.out.println("Liczba jest ujemna");
        } else {
            System.out.println("Liczba jest zerem");
        }
    }
}

2. Pętle

Pętle pozwalają na wielokrotne wykonanie tego samego bloku kodu, dopóki określony warunek jest prawdziwy. To bardzo przydatna konstrukcja, gdy chcemy powtarzać zadania, np. iterować po elementach kolekcji. Oto podstawowe pętle w Javie:

  • Pętla for:
    for (inicjalizacja; warunek; inkrementacja) { /* kod */ }
  • Pętla while:
    while (warunek) { /* kod */ }
  • Pętla do-while:
    do { /* kod */ } while (warunek);


    • Przykład: 
public class PetleExample {
    public static void main(String[] args) {
        // Petla for
        for (int i = 0; i < 5; i++) {
            System.out.println("Petla for: " + i);
        }

        // Petla while
        int j = 0;
        while (j < 5) {
            System.out.println("Petla while: " + j);
            j++;
        }

        // Petla do-while
        int k = 0;
        do {
            System.out.println("Petla do-while: " + k);
            k++;
        } while (k < 5);
    }
}

3. Instrukcje sterujące

Instrukcje sterujące takie jak break i continue pozwalają na bardziej zaawansowaną manipulację przepływem w pętlach. Mogą być używane do przerywania działania pętli lub pomijania niektórych jej iteracji.

  • Instrukcja break:
    break;
  • Instrukcja continue:
    continue;


    • Przykład: 
public class SterowanieExample {
    public static void main(String[] args) {
        // Petla z uzyciem break
        for (int i = 0; i < 5; i++) {
            if (i == 3) {
                break;
            }
            System.out.println("Break Example: " + i);
        }

        // Petla z uzyciem continue
        for (int i = 0; i < 5; i++) {
            if (i == 3) {
                continue;
            }
            System.out.println("Continue Example: " + i);
        }
    }
}

4. Iteratory

Iteratory są specjalnymi obiektami, które umożliwiają iterację (przechodzenie) po elementach kolekcji, takich jak listy czy zestawy danych. Użycie iteratora jest szczególnie przydatne, gdy chcemy bezpiecznie usunąć elementy podczas iteracji lub gdy operujemy na bardziej skomplikowanych strukturach danych. Każda kolekcja w Javie dostarcza metody do pobrania iteratora.

  • Inicjalizacja iteratora:
    Iterator<Type> iterator = kolekcja.iterator(); - tworzony jest iterator dla danej kolekcji. Iterator zaczyna swoją pracę od pierwszego elementu kolekcji.
  • Iteracja po elementach:
    while (iterator.hasNext()) { Type element = iterator.next(); /* kod */ } - metoda hasNext() sprawdza, czy w kolekcji znajdują się jeszcze elementy, a next() pobiera kolejny element.


    • Przykład: 
import java.util.ArrayList;
import java.util.Iterator;

public class IteratoryExample {
    public static void main(String[] args) {
        // Tworzenie nowej ArrayList
        ArrayList<String> lista = new ArrayList<>();
        
        // Dodawanie elementow do listy
        lista.add("element1");
        lista.add("element2");

        // Inicjalizacja iteratora
        Iterator<String> iterator = lista.iterator();
        
        // Iterowanie po elementach listy za pomoca iteratora
        while (iterator.hasNext()) {
            // Pobieranie kolejnego elementu
            String element = iterator.next();
            System.out.println("Iteracja: " + element);
        }
    }
}
