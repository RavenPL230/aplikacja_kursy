1. Definicja i deklaracja metod

Metoda to blok kodu, który wykonuje określone zadania i może zwracać wynik. Oto przykłady składni:

  • Składnia metody:
    public typZwracany nazwaMetody(parametry) { /* kod */ }
  • Przykład metody bez parametrów: 
public class MetodaExample {
    public static void main(String[] args) {
        // Wywolanie metody bezparametrowej
        przywitaj();
    }
    
    // Definicja metody bezparametrowej
    public static void przywitaj() {
        System.out.println("Witaj w programie!");
    }
}
  • Przykład metody z parametrami: 
public class MetodaExample {
    public static void main(String[] args) {
        // Wywolanie metody z parametrem
        powiedzCzesc("Bartosz");
    }
    
    // Definicja metody z parametrem
    public static void powiedzCzesc(String imie) {
        System.out.println("Czesc, " + imie + "!");
    }
}

2. Parametry metod

W Javie prymitywne typy danych są przekazywane przez wartość, a obiekty przez referencję. Oto przykład:

public class ParametryExample {
    public static void main(String[] args) {
        int liczba = 5; // Deklaracja zmiennej prymitywnej
        zwiekszLiczbe(liczba); // Wywolanie metody
        System.out.println("Liczba po wywolaniu metody: " + liczba);
    }
    
    // Metoda zmieniajaca wartosc przekazanej zmiennej
    public static void zwiekszLiczbe(int x) {
        x = x + 1; // Zwiekszenie zmiennej
    }
}

Wynik pokazuje, że oryginalna zmienna liczba się nie zmienia, ponieważ jest przekazywana przez wartość.

3. Zmienne globalne i lokalne

Zmienne globalne są dostępne w całym programie, podczas gdy lokalne mają ograniczony zakres. Oto przykład:

public class ZmienneExample {
    // Zmienna globalna
    static int globalna = 10;
    
    public static void main(String[] args) {
        // Zmienna lokalna
        int lokalna = 5;
        System.out.println("Globalna: " + globalna);
        System.out.println("Lokalna: " + lokalna);
    }
}

4. Metody przeciążone i generics

W Javie można przeciążać metody, czyli definiować kilka metod o tej samej nazwie, ale różnej liczbie lub typach parametrów. Oto przykład:

public class PrzeciazenieExample {
    public static void main(String[] args) {
        // Wywolanie metod z roznymi parametrami
        System.out.println(suma(2, 3)); // Wywolanie metody dla int
        System.out.println(suma(2.5, 3.5)); // Wywolanie metody dla double
    }
    
    // Przeciazona metoda dla typow int
    public static int suma(int a, int b) {
        return a + b;
    }
    
    // Przeciazona metoda dla typow double
    public static double suma(double a, double b) {
        return a + b;
    }
}

5. Rekurencja

Rekurencja to technika, w której metoda wywołuje samą siebie, np. do obliczenia silni:

public class RekurencjaExample {
    public static void main(String[] args) {
        // Wywolanie metody rekurencyjnej
        int wynik = silnia(5);
        System.out.println("Silnia 5 wynosi: " + wynik);
    }
    
    // Metoda obliczajaca silnie
    public static int silnia(int n) {
        if (n == 0) {
            return 1; // Warunek bazowy
        } else {
            return n * silnia(n - 1); // Rekurencyjne wywolanie metody
        }
    }
}
